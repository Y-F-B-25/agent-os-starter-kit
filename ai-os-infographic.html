Your Toolkit

What Each Tool Does (Plain English)

Seven starter-kit pieces, each with a clear job. Optional upgrades can come later.

S

START_HERE.md

Setup Co-pilot Prompt

The first message you paste into Cowork, Claude Code, Codex, or a blended lead. It runs the diagnostic, chooses a lane, and starts the setup quests.

O

Vault Folder

Durable Memory

A folder of markdown files that agents read and write. Every agent reads BRAIN_INDEX on boot to learn where everything is. Handoffs, status files, project notes, and decision logs live here.

N

Local Dashboard

Human View

A local HTML dashboard mirrors `onboarding-progress.json` so you can scan quests, blockers, handoff status, Vault Lint, and security review without connecting another service.

V

VaultBus

Agent Coordination

How agents talk to each other without being in the same session. A coordinator drops a task into an inbox file. When that agent boots, it checks the inbox and finds the task waiting.

W

Vault Lint

Structural Check

A small Python script checks broken links, ambiguous links, backticked markdown registrations, and problem orphans. It fails loudly when the vault path is wrong.

R

Local Retrieval

Search Starter

A no-dependency script indexes approved markdown and answers known questions with ranked file paths. It excludes private folders and secret-shaped content by default.

F

Optional Watcher

Cowork Bridge

A conservative Python bridge for chat-first setups that cannot run local checks directly. It accepts argv-only commands and asks you to approve each one in the terminal.

The Setup Journey

9 Quests from Zero to Running System

This is the sequence your setup co-pilot walks through. Each quest leaves a visible artifact or check.

START_HERE.md Entry Point

You paste START_HERE.md into Cowork, Claude Code, Codex, or a blended setup lead. The co-pilot runs the diagnostic, recommends a lane, writes the progress file, and prepares the dashboard.

Vault Foundation Co-pilot

The co-pilot creates or verifies the vault folders, BRAIN_INDEX, session registry, and the public docs or operations files that the generated vault references.

What you'll have: a vault with the same folders described in README, PLAYBOOKS, and BRAIN_INDEX.

Coordination Layer VaultBus

The co-pilot creates VaultBus status, command, escalation, event, and view folders. This gives agents a durable way to coordinate across sessions.

Dashboard Human View

The co-pilot updates a copied local HTML dashboard from your progress JSON. Notion is optional, and only added if you want a connected coordination hub.

Chief of Staff Prompt Boot

The co-pilot writes the first admin or Chief of Staff boot prompt. The prompt points to BRAIN_INDEX, VaultBus, and the active progress file.

Key insight: If it's not in BRAIN_INDEX, it doesn't exist to agents. This is why "Create + Register = Atomic" is a core rule.

Specialist Agent First Team

The co-pilot writes one specialist prompt based on your real goal. The default setup proves a two-agent loop before adding more.

Handoff Loop Memory Test

The agent writes a handoff and next boot prompt. A fresh session reads the handoff and can name the current task, completed work, blocker, and next action.

Quality Checks Verification

The co-pilot runs Vault Lint when local commands are available, checks onboarding and vault evals, and reports pass, warnings, or fail clearly.

Magic moment: the system can tell you what is active from the vault instead of relying on old chat memory.

Optional Power Tools Full System

Add retrieval, security sweep, optional watcher, Notion dashboard, automations, or a blended second lane only when useful.

You've graduated when: the vault exists, two prompts exist, the handoff loop works, progress is current, and you can explain where memory lives.
Setup Profiles

Which Starting Point Fits?

The diagnostic uses these profiles to choose pacing, examples, and the first setup lane.

Organizer

Cowork often fits first

You have many scattered AI chats and want continuity. The magic moment is seeing one agent resume from the vault instead of old chat.

Deep Worker

Claude Code or blended often fits

You already use AI heavily and want less overhead. The magic moment is lower-friction setup plus durable handoffs.

Enterprise Operator

Cowork plus early security fits

You need repeatable rollout and governance. The magic moment is a setup trail someone else can inspect.

Builder

Claude Code or Codex often fits

You want to edit files, repos, or systems and run checks. The magic moment is a patch verified by local tests.

Team Lead

Blended or Cowork first often fits

You want agents assigned to workstreams. The magic moment is clear ownership before anyone edits shared files.

Why It Doesn't Break

The Rules That Keep It Clean

Without governance, multi-agent systems degrade. These rules prevent that. Each one exists because something went wrong without it.

01

Context Ceiling

Agents watch the best context signal available. Real meters win. Tool calls are only a fallback. They write handoffs before they degrade.

02

Create + Register = Atomic

Any new file must be registered in BRAIN_INDEX in the same action. If it's not indexed, it doesn't exist to the next agent.

03

Zero Mechanical Work

You make decisions. Agents handle file management, checks, coordination, and records.

04

Versioned Handoffs

When an agent finishes, it writes a structured handoff. The next version boots with full context. Knowledge compounds.

05

Vault Lint

Automated checks catch orphaned files, broken references, and contradictions. The knowledge base stays clean as it grows.

06

Approved Local Execution

Local commands are verified through the active lane. Cowork can use an approval-required watcher. Claude Code and Codex can run local checks directly when the user approves the work.
